# 视频流水线 Skill V1.0

这是一组围绕短视频生产流程整理的 Skill 配置说明，用来展示如何把「选题、脚本、审核」拆成可编排的工作流节点。

适合的使用方式是：

- 作为 Skill 结构参考
- 作为 OpenClaw 工作流的起始版本
- 按自己的项目路径、通知方式和记忆字段做二次修改

授权方式见仓库根目录 [LICENSE](LICENSE)。

## 文件说明

- `video_pipeline_skill.yaml`
  单文件版本。适合快速理解从已确认选题到进入审核阶段的最短链路。

- `topic_hunter.yaml`
  选题 Skill 示例。负责扫描热点、整理候选题目。

- `script_writer.yaml`
  脚本 Skill 示例。负责根据已确认选题生成口播脚本，并衔接后续流程。

- `video_publisher.yaml`
  发布 Skill 示例。负责接收成片结果、发送审核通知并等待人工确认。

## 工作流思路

整条链路可以理解为：

`选题确认 -> 脚本生成 -> 视频流水线执行 -> 审核通知`

如果只想先跑通一版，可以直接从 `video_pipeline_skill.yaml` 开始。

如果后续需要更清晰的职责拆分，再把它拆成 `topic_hunter`、`script_writer`、`video_publisher` 三个独立 Skill。

## 使用前需要调整的内容

接入自己的环境前，建议优先检查并修改这些内容：

1. 本地项目目录
2. `.env` 的加载方式
3. `shell_exec` 实际执行的命令
4. Telegram 通知方式
5. Memory 的读写字段
6. 触发事件名称

## 适合什么场景

这组 Skill 更适合下面几类场景：

- 已经有本地视频生成脚本，想接入 Agent 编排
- 想先跑通一个粗粒度的视频工作流
- 想把内部视频流程整理成更容易复用的 Skill 结构

## 建议

如果是第一次搭这类流程，建议先用单文件版本验证链路是否通顺，再逐步拆成多 Skill 结构。这样更容易定位问题，也更容易根据自己的工具栈做替换。
